# Architecture Plan — Project Reorganization + Release Build

## Current State

```
packages/rust/
├── scarllet-proto/      (library)
├── scarllet-sdk/        (library)
├── scarllet-core/       (binary — daemon)
├── scarllet-tui/        (binary — TUI client)
├── scarllet-llm/        (library)
├── scarllet-chat-agent/ (binary — agent)   ← to move
└── tools/               (empty — ready for future tools)
```

Core watches only: `%APPDATA%/scarllet/{commands,tools,agents}/`
TUI launches: `scarllet-core.exe` from its own directory.
No release build. No local module dirs.

## Target State

```
packages/rust/
├── scarllet-proto/      (library — unchanged)
├── scarllet-sdk/        (library — unchanged)

├── scarllet-llm/        (library — unchanged)
├── agents/
│   └── default/         (binary — default agent, renamed from scarllet-chat-agent)
└── tools/
    └── echo-tool/       (binary — moved from packages/rust/echo-tool/)
```

### Release folder (generated by build target)

```
release/
├── core.exe             ← from scarllet-core
├── tui.exe              ← from scarllet-tui
├── agents/
│   └── default.exe      ← from agents/default
├── tools/
│   └── echo-tool.exe    ← from tools/echo-tool
└── commands/            ← empty for now, ready for future commands
```

### Core watch directories (6 total)

```
Local (sibling to core.exe):     User extensions (%APPDATA%/scarllet/):
  ./agents/                        %APPDATA%/scarllet/agents/
  ./tools/                         %APPDATA%/scarllet/tools/
  ./commands/                      %APPDATA%/scarllet/commands/
```

Priority: User dir overrides local. If same agent name in both, user version wins.

## Component Changes

| Component | Change |
|-----------|--------|
| `Cargo.toml` (root) | Update workspace members paths |
| `packages/rust/agents/default/` | New location for chat agent. Crate name: `default-agent`. Manifest name: `"default"` |
| `packages/rust/tools/echo-tool/` | Moved from `packages/rust/echo-tool/`. Crate name unchanged. |
| `scarllet-core/src/watcher.rs` | Add local dirs (relative to binary). Handle override priority. |
| `scarllet-tui/src/main.rs` | `spawn_core` looks for `core` (not `scarllet-core`) as sibling |
| Root `project.json` | New `release` target: `cargo build --release` + copy script |
| `scripts/release.ps1` | PowerShell script to copy binaries to `release/` |

## Implementation Order

1. **Move echo-tool** to `packages/rust/tools/echo-tool/` + update Cargo.toml members + project.json
2. **Move chat agent** to `packages/rust/agents/default/` + rename crate + update Cargo.toml members + project.json
3. **Update Core watcher** — add local dirs, handle override priority
4. **Update TUI spawn_core** — look for `core` (sibling binary name)
5. **Create release build target** — root project.json + copy script
6. **Verify** — build, release, test the full flow

## Principles Applied

| Principle | Application |
|-----------|-------------|
| **SRP** | Watcher owns discovery from all dirs. Release script owns the copy layout. Each concern is in one place. |
| **OCP** | Adding new agents/tools means adding a crate under `agents/` or `tools/` — no Core or TUI code changes needed. The release script discovers binaries by convention. |
| **DIP** | Core resolves dirs from its own binary path — no hardcoded project paths. Portable. |
| **KISS** | Simple PowerShell script for release, not a plugin or framework. Convention-based folder structure. |
| **DRY** | One watcher function handles both local and user dirs. One release script handles all binary types. |

## Risks

| Risk | Mitigation |
|------|-----------|
| Git doesn't track moves well with content changes | Do the move in a separate commit from code changes |
| echo-tool path changes break existing project.json refs | Update all $schema relative paths |
| Binary name mismatch after move | Verify `cargo build` produces expected binary names |
